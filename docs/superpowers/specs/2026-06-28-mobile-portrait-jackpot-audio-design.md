# Mobile Portrait Jackpot Audio Design

**Goal**

为当前 Godot 生存游戏新增三连拉霸全拿结算、移动端波次震动、轻量程序音效，并把移动端体验从横屏切到竖屏，完成一版以“爽感优先”为目标的 UI 适配；同时定义后续可玩性扩展的第一阶段方案。

**Project Context**

- 当前项目已经有拉霸系统、升级 3 选 1、移动摇杆、波次提醒、羁绊/武器成长。
- 当前 `project.godot` 仍然是 `1280x720` 横屏视口，HUD 节点大多按横屏绝对坐标布局。
- 当前移动端重点问题是：竖屏未适配、拉霸大奖仍是 6 选 1、战斗缺少声音与手机震动反馈。

## Scope

### In Scope

1. 拉霸三连相同改为“6 个奖励全部拿走”
2. 拉霸大奖表现改为 A3：先显示大奖，再连续自动结算
3. 每波开始加入移动端手机震动
4. 武器加入基础程序音效
5. 怪物死亡加入基础程序音效
6. 将移动端改为竖屏布局，并完成 HUD 适配
7. 明确下一阶段可玩性增强方案（P1 + P2）

### Out of Scope

1. 局外养成、存档、角色选择
2. 大规模美术资源替换
3. 完整音频资源包接入
4. 所有事件/构筑内容一次性全部实现

## Recommended Approach

采用“轻量高反馈”方案：

- 奖励逻辑不再让玩家做额外选择，减少停顿，直接给足爽感
- 震动只绑定高价值节点（波次、Boss、Jackpot），避免耗电和干扰
- 音效先用程序短音实现，不额外引入音频包，控制包体和性能
- 竖屏适配优先通过 HUD 重排 + 摄像机/信息密度调整完成，避免战斗层大改

## Design

### 1. Jackpot Reward Flow

当拉霸三个符号完全一致时：

- HUD 显示 `JACKPOT / 全拿！`
- 不再显示“6 选 1”的按钮交互
- 进入自动结算队列，按约 `0.10s` 的间隔逐个发放 6 个奖励
- 每发放一个奖励：
  - HUD 对应卡牌点亮/闪烁
  - 播放短促 Jackpot 音效
  - 可选触发一次轻微镜头震动
- 6 个奖励全部结算完成后：
  - 关闭拉霸 overlay
  - 恢复战斗

这样兼顾“我全都要”的爽感和可读性，不会让玩家手动点 6 次，也不会瞬间刷完看不清。

### 2. Mobile Haptics

只在移动端启用手机震动：

- 普通波次开始：短震一次
- 第 10 / 20 波：中等震动
- 第 30 波或 Boss 来袭：更长、更明显的双段震动
- Jackpot 命中：短促高频震动一次

实现原则：

- 不对普通攻击、普通击杀逐次震动
- 统一通过一个小型触觉辅助方法触发，集中做平台判断和节流
- 桌面端与无触觉平台自动降级为空操作

### 3. Audio Design

加入统一的轻量 `sfx_manager`，负责生成并播放程序音效。

首批音效覆盖：

- 武器发射
  - 弹道类：脆短发射音
  - 激光类：高频扫射音
  - 爆炸类：低频闷爆音
  - 环绕/触手类：暗色挥扫音
  - 僚机类：机械灵体脉冲音
- 敌人死亡
  - 普通怪：短爆裂
  - 精英怪：更重的爆裂
  - Boss：单独较长爆音
- Jackpot 结算：短促中奖音
- 波次提醒：极短 UI 提示音

实现原则：

- 程序生成 + 缓存 `AudioStreamWAV`
- 统一节流，避免高频同类音效叠得太吵
- 移动端默认音量略低于桌面端战斗音

### 4. Portrait Mobile Layout

移动端目标改为竖屏版本：

- 视口基准改为 `720x1280`
- 手持方向改为竖屏
- HUD 重排：
  - 顶部：血条 / 波次 / 关键信息
  - 中上：武器与遗物摘要
  - 中部：波次警告 / Jackpot 演出
  - 底部左：摇杆
  - 底部右或底部中：提示文本与可能的交互按钮
- 升级面板改为更适合竖屏的 1 列或 2 列布局
- 拉霸界面改为竖向居中展示，结果区和机器视觉对齐

战斗层配合：

- 摄像机在移动端适当拉高一点可视范围，但保持竖屏可读性
- HUD 文本长度控制，避免右上角信息在窄屏溢出

### 5. Phase-2 Replayability Direction

本轮只先把入口和设计定好，后续优先级如下：

#### P1 局内随机事件

- 恶魔交易：牺牲血量/属性，换高价值奖励
- 精英悬赏：刷指定精英，击杀给奖励
- 临时祝福：持续 1 波的强力正面效果

#### P2 构筑强化

- 武器专精：连续培养同类武器时，提高相关升级权重
- 羁绊机制强化：部分羁绊从纯数值升级为机制变化
- 终极词条：合体武器附带额外玩法效果

## File-Level Impact

### Modify

- `project.godot`
  - 切换竖屏默认视口与移动方向
- `scripts/ui/hud.gd`
  - Jackpot 全拿状态机
  - 竖屏 HUD 布局逻辑
  - Jackpot 自动结算演出
- `scenes/ui/HUD.tscn`
  - 竖屏节点排布
  - 升级面板与拉霸面板位置尺寸调整
- `scripts/game/game.gd`
  - Jackpot 自动奖励发放
  - 波次震动触发
  - 音效触发点接入
- `scripts/weapons/weapon_manager.gd`
  - 提供 Jackpot 6 奖励完整结算支持
  - 为武器提供音效分类或标识

### Create

- `scripts/audio/sfx_manager.gd`
  - 程序音效生成、缓存、播放、节流

## Error Handling

- 若平台不支持震动，直接忽略，不报错
- 若 Jackpot 奖励数量异常，退化为普通升级面板展示
- 若音效生成失败或被禁用，战斗逻辑不受影响
- 若移动端尺寸异常，HUD 至少保持关键血条/摇杆/升级面板可用

## Testing Strategy

1. 无头编译验证
2. 主场景 5 秒冒烟验证
3. Jackpot 三连场景手动检查：
   - 是否自动发放 6 个奖励
   - 是否不再停留在“6 选 1”
4. 移动端布局手动检查：
   - 竖屏下血条、波次、摇杆、升级面板是否可见
5. 音效节流手动检查：
   - 大量弹道武器时不应爆音失控
6. 波次震动手动检查：
   - 普通波、10/20 波、Boss 波反馈有强弱区别

## Success Criteria

- 三连拉霸时，6 个奖励全部自动结算
- 移动端进入游戏默认竖屏
- 竖屏下 HUD 可读、摇杆可用、升级界面不遮挡核心信息
- 波次开始时移动端能感知到手机震动
- 武器攻击与怪物死亡具备基本听觉反馈
- 不引入明显卡顿或新的阻塞错误
